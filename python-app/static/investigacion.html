DMS + HRMS

Sistemas de Gestión Documental y RRHH
Comparativa para el Archivo Digital UCV

Módulo Archivo — DMS institucional Módulo RRHH — DMS adaptado

Julio 2026

El Archivo Digital Ciencias UCV opera dos módulos distintos en un mismo sistema: uno es un DMS institucional pleno (documentos, tipologías, FTS, ISAD(G)), y el otro es un DMS adaptado para RRHH (expediente por empleado, partes I–IV, alertas laborales, LOTTT). Esta comparativa cubre ambas familias de sistemas.

Sistemas DMS / ECM — Gestión Documental Institucional

Alfresco Community Más cercano
ECM Open Source · Java · Apache Solr

El referente en ECM open-source empresarial. Ofrece repositorio de contenido, versionado, workflows BPMN, metadatos configurables, FTS con Solr y API REST. La edición Community es gratuita; Enterprise tiene soporte pagado.

Funcionalidad: 9.5 Facilidad deploy: 3.5 Relevancia UCV: Alta
DSpace
Repositorio Institucional · Java · Solr

El repositorio digital más extendido en universidades (MIT, Cambridge, UCV propia lo usa para tesis). Especializado en preservación a largo plazo, Dublin Core / MARC21, OAI-PMH para interoperabilidad. No es un DMS generalista.

Preservación: 9.5 Gestión doc: 5.5 Relevancia UCV: Media
OpenKM Community
DMS Open Source · Java · Hibernate

Orientado a PYMEs y administración pública. Incluye versionado, workflow simple, categorías, FTS con Lucene y control de acceso. Interfaz más ligera que Alfresco; administración más accesible para equipos pequeños.

Funcionalidad DMS: 7.5 Facilidad deploy: 6.0 Relevancia UCV: Media-Alta
Paperless-ngx Inspira UX
Open Source · Python/Django · PostgreSQL

DMS personal/pequeño equipo. Ingesta automática de PDFs (OCR con Tesseract), etiquetas, FTS, versionado. Stack Python + PostgreSQL — tecnológicamente muy cercano a la arquitectura UCV. Interfaz limpia y minimalista.

UX/Usabilidad: 9.0 Escala inst.: 4.0 Relevancia UCV: Alta (Ref)

Sistemas HRMS / HRIS — Gestión Documental de RRHH

OrangeHRM Open Source Más cercano
HRMS Open Source · PHP · MySQL

HRMS completo de código abierto. La edición Community cubre el 80% de necesidades de una institución pública. Usado en universidades latinoamericanas. Módulo "Documents" maneja exactamente el concepto de "partes" de expediente.

Gestión exp.: 8.5 Legislación VE: Baja Relevancia UCV: Alta
Odoo HR (v17)
ERP Open Source · Python · PostgreSQL

Módulo de RRHH dentro del ERP Odoo. Stack Python + PostgreSQL idéntico al sistema UCV. Gestiona empleados, contratos, permisos, documentos. Versionado de contratos y alertas de vencimiento incluidos. Extensible.

Gestión doc: 7.0 Stack alignment: Alta Alertas/Vencim: 8.5
SIGEP / SigePlus (Venezuela)
Sistema público venezolano · ONAT

Sistema oficial del Estado venezolano. UCV como autónoma no está obligada a usarlo, pero define los campos estándar del expediente: datos personales, cargos, títulos, sanciones. Referente normativo para RRHH UCV.

Relevancia normativa: Alta Tecnología: Obsoleta
BambooHR Inspira UX
SaaS · HRIS para equipos medianos

Destaca por su UX para expedientes digitales: drag-and-drop de documentos, carpetas por categoría de documento (análogas a las Partes I–IV), versionado automático, alertas. Referente de experiencia de usuario.

UX expediente: 9.0 Versionado docs: 8.5 Relevancia UCV: Ref. UX

Lecciones Aplicables al Sistema UCV

De Alfresco: Metadatos extensibles

El sistema UCV hardcodea los campos ISAD(G). Un panel de admin para campos personalizados por tipo de documento daría mucha más flexibilidad.

Aplicabilidad: Módulo Archivo — media prioridad, requiere refactoring de modelo de datos.
De Paperless-ngx: OCR automático

Monitorea y procesa automáticamente PDFs escaneados: OCR, clasificación. Un worker similar podría procesar archivos subidos a R2 y extraer texto para FTS.

Aplicabilidad: Ambos módulos — alta prioridad para accesibilidad, requiere worker separado.
De BambooHR: UX por categorías

Organiza documentos en pestañas análogas a las Partes I–IV. Su UX de carga (drag-and-drop directo en la sección de la parte) es más intuitivo que el modal actual.

Aplicabilidad: Módulo RRHH — media prioridad. El dossier podría incluir subida directa por sección.
De Odoo HR: Alertas de vencimiento activas

Envía notificaciones cuando documentos están próximos a vencer. UCV tiene las fechas pero falta envío automático (email o banner al admin).

Aplicabilidad: Módulo RRHH — alta prioridad. Implementable como cron o banner en dashboard.
De SIGEP: Campos normativos de expediente

Define modelo de datos público venezolano. El sistema UCV cubre LOTTT básica pero aún no tiene títulos académicos o evaluaciones de desempeño (auditoría ONAT).

Aplicabilidad: Módulo RRHH — alta relevancia normativa. Agregar tabla titulos_empleados.
